# 🟡 Уровень 2 — Когортный анализ и средний LTV

**Для продолжающих:** Анализ LTV по когортам пользователей  

⏱ **Рекомендуемое время:** 30–40 минут  
🎯 **Цель:** Научиться рассчитывать LTV по когортам пользователей и средний LTV

---

## 📌 Задание 1 — LTV по пользователю с когортами

**Цель:** Рассчитать LTV каждого пользователя вместе с месяцем его когорты.  

**Инструкции:**

1. Используйте таблицы `orders` и `users`  
2. Учитывайте только **не возвращённые заказы**  
3. Прибыль = `revenue - cost`  
4. Назначьте каждому пользователю **месяц когорты** по `signup_date`  
5. Группируйте по `user_id` и `cohort_month`  
6. Сортируйте по `cohort_month` и LTV в порядке убывания  

**Подсказка:**  
- Используйте `toStartOfMonth(u.signup_date)` для месяца когорты  
- Соедините таблицы `orders` и `users` по `user_id`

---

## 📌 Задание 2 — Средний LTV по когортам

**Цель:** Рассчитать **средний и медианный LTV** по каждому месяцу когорты.  

**Инструкции:**

1. Используйте расчёт LTV по пользователю из Задания 1  
2. Группируйте по `cohort_month`  
3. Рассчитайте:
   - Средний LTV: `AVG(ltv)`  
   - Медианный LTV: `quantile(0.5)(ltv)`  
4. Сортируйте по `cohort_month` по возрастанию  

**Подсказка:**  
- Можно использовать **CTE или подзапрос**, чтобы сначала посчитать LTV по пользователям  
- Медианный LTV помогает нивелировать выбросы

## 📌 Задание 3 — Выводы по когортам (письменно)

Ответьте на следующие вопросы:

1. Какая когорта имеет наибольший **средний LTV**?  
2. Есть ли когорты с необычно **высоким или низким LTV**? Почему?  
3. Как возвраты заказов могут повлиять на LTV на уровне когорты?  
4. Почему иногда **медианный LTV** лучше среднего?
