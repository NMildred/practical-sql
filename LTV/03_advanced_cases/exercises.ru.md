# 🔵 Уровень 3 — Retention × LTV и Predictive LTV

**Для продвинутых:** Построение матриц retention и прогнозирование LTV  

⏱ **Рекомендуемое время:** 40–50 минут  
🎯 **Цель:** Понять динамику удержания пользователей и прогнозировать LTV

---

## 📌 Задание 1 — Матрица retention по когортам

**Цель:** Рассчитать retention пользователей по когортам и месяцам жизни.  

**Инструкции:**

1. Используйте таблицы `orders` и `users`  
2. Определите месяц когорты по `signup_date`  
3. Рассчитайте **месяц жизни** для каждого заказа:  
   - `lifetime_month = dateDiff('month', cohort_month, order_month)`  
4. Посчитайте **количество уникальных пользователей**, оставшихся в каждом месяце жизни  
5. При желании рассчитайте **коэффициент retention** по когорте  

**Подсказка:**  
- Используйте **CTE cohort_sizes**, чтобы узнать размер когорты  
- Retention rate = `retained_users / cohort_size`

---

## 📌 Задание 2 — LTV на удержанного пользователя

**Цель:** Для каждой когорты и месяца жизни рассчитать **средний LTV на удержанного пользователя**.  

**Инструкции:**

1. Суммируйте прибыль на удержанного пользователя: `revenue - cost` (только не возвращённые заказы)  
2. Разделите накопленную прибыль на количество удержанных пользователей  
3. Группируйте по `cohort_month` и `lifetime_month`  

**Подсказка:**  
- Объединяет retention и LTV в одну матрицу  
- Отлично подходит для **визуализации в виде heatmap**

---

## 📌 Задание 3 — Прогнозный LTV

**Цель:** Сделать прогноз будущего LTV по когортам на основе исторических данных.  

**Инструкции:**

1. Используйте исторический LTV по пользователю и по когорте  
2. Постройте **накопительный LTV** по месяцам жизни  
3. Примените простой прогнозный метод:
   - Например, экспоненциальное затухание:  
     `predictive_ltv = avg_ltv * exp(-0.1 * lifetime_month)`  
4. Сравните накопительный и прогнозный LTV  

**Подсказка:**  
- Даже простая формула показывает **затухание удержания**  
- В продакшене её можно улучшить статистическими или ML-моделями
