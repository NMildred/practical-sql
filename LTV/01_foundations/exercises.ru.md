# 🟢 Уровень 1 — Основы

**Для начинающих:** Расчёт исторического LTV по пользователю  

⏱ **Рекомендуемое время:** 20–30 минут  
🎯 **Цель:** Научиться вычислять чистую прибыль (LTV) для каждого пользователя на данных, приближённых к продакшену

---

## 📌 Задание 1 — Исторический LTV по пользователю

**Цель:** Рассчитать чистую прибыль для каждого пользователя.  

**Инструкции:**

1. Используйте таблицу `orders`  
2. Учитывайте только **не возвращённые заказы** (`refunded = 0`)  
3. Прибыль = `revenue - cost`  
4. Группируйте по `user_id`  
5. Сортируйте по LTV в **убывающем порядке**  

**Подсказка:**  
- Используйте `SUM(if(refunded = 0, revenue - cost, 0))`  

---

## 📌 Задание 2 — Топ 5 пользователей по LTV

**Цель:** Определить самых ценных пользователей.  

**Инструкции:**

1. Используйте расчёт LTV из Задания 1  
2. Сортируйте в порядке убывания  
3. Ограничьте результат **топ-5 пользователей**

**Подсказка:**  
- Добавьте `LIMIT 5` в конце запроса  
- Проверьте, что сортировка идёт по LTV в убывающем порядке  

---

## 📌 Задание 3 — Проверка данных (письменно)

Ответьте на следующие вопросы:

1. Есть ли пользователи с **LTV = 0**? Почему?  
2. Есть ли пользователи с **отрицательной прибылью**? Как это может произойти?  
3. Как **возвраты заказов** влияют на общий LTV?  
4. Какие **возможные проблемы** могут быть в этом расчёте LTV?  

**Подсказка:**  
- Подумайте о пользователях без заказов  
- Учтите случаи, когда `cost > revenue`  
- Обратите внимание на выбросы и возвраты
